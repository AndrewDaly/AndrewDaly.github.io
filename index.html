AutoHotKey is a powerful scripting language to automate scripts and create custom hotkeys.


I was inspired by almogtavor's script below, and created my own AHK version two script

In my opinion, there is a fundamental disconnect between how users input actions and how computer applications receive them

User's will do whatever is most familiar and intuitive, rather than what is necessarily optimal. This is no fault of their own and is often a non-issue

Likewise, applications are developed to be easily utilized, by the largest possible audience

Deviating from a standard mouse and keyboard setup, thereby can limit the potential market for a system

So it is rational for developers to make software that caters to a mouse and keyboard. It is the Nash Equilibibrium point of user interactions

That doesn't mean it is the best equilibrium state for users or applications

Shifting such equilibriums often require almost societal level changes, so it's not feasible to count on it changing any time soon

By using AutoHotKey, you can shift the balance into a more keyboard centric experience

My version is centered around making standard hotkeys more accessible. Essentially, redirecting the 'j' key, to be the 'ctrl' key

The 'j' letter, is one of least used letters in then english language https://en.wikipedia.org/wiki/Letter_frequency

And most hotkeys are centered around the 'ctrl' key. The 'j' key sits on homerow in most computers, whereas 'ctrl' is far to the side

By making hotkeys easier to use, you don't need to switch to the mouse as often, improving efficiency

This isn't only an ease of use and efficiency thing, it is an accessibility one too
